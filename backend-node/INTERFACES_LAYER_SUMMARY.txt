🚀 FLUJO DE SOLICITUD HTTP:

   1. Cliente: GET /api/auth/me
              Authorization: Bearer eyJhbGc...
                         ↓
   2. Express: app.use('/api/auth', createAuthRoutes(authService))
                         ↓
   3. Middleware: requireAuth(authService)
      ├─ Extrae token del header
      ├─ authService.verificarToken(token)
      ├─ Valida JWT (firma, expiración, issuer, audience)
      ├─ ✅ Válido: inyecta req.user = { uid, email, role, permisos[] }
      └─ ❌ Inválido: retorna 401
                         ↓
   4. Middleware: requirePermissions(...permisos)  [Opcional]
      ├─ Verifica req.user.permisos incluye TODOS los requeridos
      ├─ ✅ Tiene permisos: continúa
      └─ ❌ Sin permisos: retorna 403
                         ↓
   5. Controller: AuthController.getMe(req, res)
      ├─ Accede a req.user (inyectado por requireAuth)
      ├─ Retorna datos del usuario autenticado
      └─ res.json({ success: true, data: { uid, email, role, permisos } })
                         ↓
   6. Cliente: Recibe JSON con datos del usuario

🔧 CÓMO INICIAR EL SERVIDOR:

   1. INSTALACIÓN:
      npm install

   2. CONFIGURAR VARIABLES DE ENTORNO:
      cp .env.example .env
      # Editar .env con credenciales reales de Firebase y JWT_SECRET

   3. DESARROLLO CON HOT RELOAD:
      npm run dev
      └─ Escucha en http://localhost:3000

   4. BUILD PARA PRODUCCIÓN:
      npm run build
      npm start

🧪 TESTING RÁPIDO:

   1. Health Check:
      curl http://localhost:3000/health

   2. Login (necesitas idToken de Google):
      curl -X POST http://localhost:3000/api/auth/login \
        -H "Content-Type: application/json" \
        -d '{"idToken":"<GOOGLE_ID_TOKEN>"}'

   3. Get User Data:
      curl -X GET http://localhost:3000/api/auth/me \
        -H "Authorization: Bearer <JWT_OBTENIDO>"

   4. Logout:
      curl -X POST http://localhost:3000/api/auth/logout \
        -H "Authorization: Bearer <JWT>"
